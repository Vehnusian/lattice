# Positioning

## What this is

A modern, searchable, mobile-friendly reference library of validated complex systems models.

Brockmann's content model, executed with Distill's visual polish bar, with the information architecture neither of them built.

## Audience

University students through professors. People who:

- Need to quickly grasp an unfamiliar model
- Reference a known model while teaching or studying
- Play with a model to build intuition
- Cite or link to it in their own work

## Use mode

Reference, not narrative. Visit for a specific thing → orient in seconds → depth on demand → leave.

## Hard rules from this positioning

1. Section ordering is identical across every model page. Visiting one page teaches the layout for every page.
2. Every page is skimmable in under 30 seconds and deeply usable in 5 minutes.
3. One main simulation per page. No fragmenting interaction into multiple inline widgets — that's narrative pacing, wrong medium.
4. Math is available but tucked. Default state shows behavior, not equations.
5. Citations are explicit and clickable. Original paper, canonical textbook, free alternative.
6. Tag taxonomy and search are first-class features, not afterthoughts.
7. Mobile works. Most academic users browse on laptops; some on phones. Both must be usable.

## What this is NOT

- A storytelling explorable (Nicky Case territory)
- A long-form essay journal (Distill territory)
- A textbook replacement
- A teaching curriculum

## Per-page template (Brockmann + NetLogo)

1. Title + one-line definition
2. Live simulation with the 2-3 parameters that actually matter
3. "How it works" — short prose, no math required
4. "Things to try" — 2-4 suggested experiments
5. Math / equations — collapsed by default
6. References — paper + textbook + free alternative
7. "Cite this page" + code link
